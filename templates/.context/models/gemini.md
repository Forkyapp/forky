# Gemini Analysis Guidelines

## Documentation Style

- Write clear, concise technical documentation
- Include architecture diagrams when relevant
- Break down complex features into steps

## Feature Specification Format

```markdown
# Feature: {Name}

## Overview
{Brief description}

## Requirements
- Requirement 1
- Requirement 2

## Implementation Steps
1. Step 1
2. Step 2

## Technical Considerations
{Architecture, dependencies, etc}
```

## Analysis Focus

- Understand task requirements thoroughly
- Identify technical challenges
- Suggest implementation approach
- Document assumptions
